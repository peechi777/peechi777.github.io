Case Study 01

銀行商品系統

以 Spring Boot 3、Vue 3、MySQL 與 Docker Compose 建構三層式銀行商品平台，重點放在 RESTful API、Stored Procedure、交易一致性與可重現部署。

Java 17Spring Boot 3Vue 3MySQL 8Docker Compose
Lab 01

問題與系統邊界

資料操作風險

銀行商品、會員收藏與交易流程需要避免 SQL Injection、錯誤更新與不一致狀態。

前後端整合

Vue 前端透過 API 與 Spring Boot 串接，讓頁面操作、驗證與資料回寫保持清楚分工。

Lab 02

RESTful API 與 MVC 分層

Controller、Service、Repository 各自負責請求入口、商業邏輯與資料存取，並透過 GET / POST / PUT / DELETE 對應商品與會員操作。

Lab 03

Stored Procedure 與交易控制

以 MySQL Stored Procedure 收斂重要資料流程，搭配 Spring 的 Transaction 管理，讓多步驟操作能正確提交或回復。

Lab 04

Vue 3 前端與 Docker 部署

前端使用 Vue 3 / Vite 管理商品列表與使用者互動；部署端以 Docker Compose 啟動 API 與資料庫，降低環境差異。
