全端容器化銀行金融商品管理系統

基於 Spring Boot 3 與 Vue 3 的前後端分離架構，結合 MySQL 預儲程序 與 Docker Compose，實現一鍵自動化部署與金融級資料安全操作。

Java 17 / Spring Boot 3 Vue 3 / Vite MySQL 8.0 Docker Compose RESTful API

開發與架構痛點

金融計算精度與資安風險

若在前端或中介層計算金融手續費與總額，容易受到竄改且存在精度丟失問題。分散的查詢邏輯亦容易引發 SQL Injection 風險。

環境部署繁瑣與運維困難

傳統 Web、API 與資料庫需要各自獨立安裝與設定環境，這導致「在我的電腦可以跑，在別人的不行」的問題，極大降低開發交付效率。

技術架構與核心實作

1. 嚴謹的 RESTful API 與 Controller 職責分離

後端採用 MVC 分層架構，確保代碼高內聚低耦合。API 遵循 RESTful 精神，利用 HTTP Method 區分操作行為。

  • 將邏輯清晰拆分為：商品展示 (BankProductController)、用戶查詢 (UserController) 與 喜好清單操作 (LikeListController)。
  • 支援以 PathVariable 與 RequestParam 精確進行 GET/POST/PUT/DELETE 資源操作。

2. 核心計算下移：MySQL 預儲程序 (Stored Procedure)

解決金融計算精準度問題，實作 「資料庫處理核心邏輯」。透過 Spring Data JPA 的 @Procedure 直接對接資料庫底層。

  • 申購總手續費與總金額由資料庫內的 SP 動態計算後直接落表，確保資料高一致性。
  • 透過 JdbcTemplate 執行參數化查詢，從根源阻斷 SQL 隱碼攻擊。

3. Vue 3 Composition API 與極速前端渲染

放棄繁重的舊版框架，擁抱 Vite + Vue 3 的輕量體系，打造即時、高互動的用戶介面。

  • 高自定義元件：將商品列表、清單渲染封裝於 ProductList.vue，自動移除未使用的冗餘樣板代碼。
  • Axios 異步整合：採用 Async/Await 語法平滑串接後段微服務，並優雅處理前端跨域請求 (CORS)。

4. Docker Compose 全容器化的一鍵部署

建立跨平台的穩定生態圈！以 PowerShell 腳本包裝編排機制，徹底抹平環境差異。

  • 自動化資料庫初始化：透過掛載 DDL 與 DML 腳本至 docker-entrypoint-initdb.d 達成啟動即建表塞資料。
  • 服務相依管理：使用 depends_on 精準控制啟動順序，確保 API Server 銜接 DB 完成。

關鍵工程實踐與系統優勢

@Transactional
事務管理與 ACID

在 Service 層的資料變更行為強制導入事務管理。面對複雜操作時，若中途發生 Exception，系統能保證資料 Rollback，絕不出現髒資料。

Zero Config
開箱即用的開發體驗

整合 Maven Wrapper，讓後續開發者或測試人員，即便電腦未安裝任何 Java 開發包與資料庫套件，也能透過一句指令完成編譯與部署。

Decoupling
前後端完全解耦

突破傳統 JSP/Thymeleaf 綁定限制，實作純 JSON 資料流交互。未來若需擴充 iOS 或 Android APP 端，後端 API 可 100% 無痛共用。
